Playback Reporter

Settings

Sends a notification through Emby's notification system when a user reports a playback issue.

How to Use

Once the client script is installed (see Action Sheet Integration below), users can report a playback problem directly from the media action sheet — no bookmarks or extra steps required.

Reporting an Issue

  1. Open any movie or episode in the Emby web client.
  2. Click the action menu (⋮ or the three-dot button) for that item.
  3. Select Report Issue from the menu.
  4. Enter a short description of the problem and confirm.

The report will appear in the Issue Reports log below, and a server notification will be sent if that option is enabled.

Bookmarklet (alternative method)

If you prefer not to install the client script, you can use the bookmarklet below as a fallback. Create a browser bookmark and paste the code into its URL field. Then navigate to the details page of a movie or episode in the Emby web client and click the bookmark to report an issue.

External API

You can retrieve the list of reported issues using an external application like Home Assistant. This allows for creating automations or custom notifications.

Authentication

API requests are authenticated using a standard Emby API key. You can generate one in your Emby Server Dashboard under API Keys. The key must be sent as a URL parameter.

GET /EmbyReporter/Public/Issues

Retrieves a list of the most recent playback issue reports (up to 200).

Example Request URL:

http://[your-emby-address]:8096/EmbyReporter/Public/Issues?api_key=YOUR_EMBY_API_KEY

Example Response:

[
    {
        "Timestamp": "2025-11-01T10:30:00Z",
        "EventType": "Playback Issue",
        "Username": "TestUser",
        "ClientName": "Chrome",
        "ItemName": "My Movie (2025)",
        "LibraryName": "Movies",
        "Description": "Audio is out of sync",
        "Path": "/data/movies/My Movie (2025)/movie.mkv"
    }
]

Action Sheet Integration

Adds a "Report Issue" option directly into the Emby media action sheet. Clicking Install replaces dashboard-ui/modules/actionsheet/actionsheet.js with the plugin's modified version (the original is backed up automatically). Clicking Remove restores the original file.

Issue Reports

Shows the last 200 reported playback issues.
